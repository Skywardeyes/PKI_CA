# GM 按钮 0-9 文件影响清单（含文件含义）

本文按 Web 控制台 `GM 国密算法专区` 的实际后端实现（`/api/gm/*`）整理。  
目标：演示时可快速说明每个按钮会**生成哪些文件、删除哪些文件、修改哪些文件**，并给出文件的简要含义。

> 说明
>
> - 以下基于“演示顺利（成功返回）”场景。
> - `clientName` 以 `trainee-gm` 为例，实际为 `client-<name>`。
> - 所有 GM 写操作按钮（0~9）都会向 `artifacts/logs/audit.jsonl` 追加一条审计记录（`profile=gm`）。
> - GM 链路运行产物默认在 `gm/` 与 `artifacts/logs/gm/` 下，与 Intl 隔离。
> - 路径均为仓库相对路径。

- `gm/ca/`（整目录删除，旧 GM CA 产物清空）

- `artifacts/logs/gm/`（整目录删除，旧 GM 验证日志清空）
- `web/app/__pycache__/`（可选删除；仅当不保留缓存）

### 修改

- 无“内容更新式”修改（本步骤是“先删后重建”）；`audit.jsonl` 为追加写入。

## 3. 签发 GM 服务端 + 客户端证书（`issue`）

### 生成

首次或 `gm/server/server.cert.pem` 不存在时会生成服务端文件：

- `gm/server/server.key.pem`（GM 服务端私钥，SM2）
- `gm/ca/intermediate/csr/server.csr.pem`（GM 服务端 CSR）
- `gm/server/server.cert.pem`（GM 服务端证书，由 GM Intermediate 签发）

每次都会生成客户端文件（以 `trainee-gm` 为例）：

- `gm/client/client-trainee-gm.key.pem`（GM 客户端私钥，SM2）
- `gm/ca/intermediate/csr/client-trainee-gm.csr.pem`（GM 客户端 CSR）
- `gm/client/client-trainee-gm.cert.pem`（GM 客户端证书，由 GM Intermediate 签发）
- `gm/client/client-trainee-gm.p12`（GM 客户端导入包，含私钥/证书/链）
- `gm/ca/intermediate/newcerts/*`（GM Intermediate 已签发证书归档副本）
- `artifacts/logs/audit.jsonl`（审计日志追加）

### 删除

- 无。

### 修改

- `gm/ca/intermediate/index.txt`（GM Intermediate 台账新增服务端/客户端签发记录）
- `gm/ca/intermediate/serial`（GM Intermediate 序列号计数器递增）
