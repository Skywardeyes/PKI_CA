# Intl 按钮 0-9 文件影响清单（含文件含义）

本文按 Web 控制台 `Intl 国际算法专区` 的实际后端实现（`/api/intl/*`）整理。  
目标：演示时可快速说明每个按钮会**生成哪些文件、删除哪些文件、修改哪些文件**，并给出文件的简要含义。

> 说明
>
> - 以下基于“演示顺利（成功返回）”场景。
> - `clientName` 以 `trainee` 为例，实际为 `client-<name>`。
> - 所有 Intl 写操作按钮（0~9）都会向 `artifacts/logs/audit.jsonl` 追加一条审计记录。
> - 路径均为仓库相对路径。

---

## 0. 演示重置（`reset-demo`）

### 生成

- `ca/root/certs/`（Root CA 证书目录）
- `ca/root/crl/`（Root CA CRL 目录）
- `ca/root/newcerts/`（Root CA 已签发证书归档目录）
- `ca/root/private/`（Root CA 私钥目录）
- `ca/root/csr/`（Root CA CSR 目录）
- `ca/intermediate/certs/`（Intermediate 证书目录）
- `ca/intermediate/crl/`（Intermediate CRL 目录）
- `ca/intermediate/newcerts/`（Intermediate 已签发证书归档目录）
- `ca/intermediate/private/`（Intermediate 私钥目录）
- `ca/intermediate/csr/`（Intermediate CSR 目录）
- `server/`（服务端证书与私钥目录）
- `client/`（客户端证书、私钥、P12 目录）
- `artifacts/logs/`（日志目录）
- `ca/root/index.txt`（Root 证书台账数据库）
- `ca/intermediate/index.txt`（Intermediate 证书台账数据库）
- `ca/root/serial`（Root 下次签发序列号计数器，初始 `1000`）
- `ca/intermediate/serial`（Intermediate 下次签发序列号计数器，初始 `2000`）
- `ca/intermediate/crlnumber`（Intermediate 下次 CRL 编号计数器，初始 `2000`）
- `artifacts/logs/audit.jsonl`（审计日志，记录本次 reset-demo）

### 删除

- `ca/root/`（整目录删除，旧 Root 产物清空）
- `ca/intermediate/`（整目录删除，旧 Intermediate 产物清空）
- `server/`（整目录删除，旧服务端证书产物清空）
- `client/`（整目录删除，旧客户端证书产物清空）
- `artifacts/`（整目录删除，旧验证/审计日志清空）
- `web/app/__pycache__/`（可选删除；仅当不保留缓存）

### 修改

- 无“内容更新式”修改（本步骤是“先删后重建”）；`audit.jsonl` 为追加写入。

## 2. 构建 Root / Intermediate CA（`build-ca`）

### 生成

- `ca/root/private/ca.key.pem`（Root CA 私钥，最高信任锚签名私钥）
- `ca/root/certs/ca.cert.pem`（Root CA 自签证书，信任锚公钥与身份声明）
- `ca/intermediate/private/intermediate.key.pem`（Intermediate CA 私钥，日常签发使用）
- `ca/intermediate/csr/intermediate.csr.pem`（Intermediate 的证书签名请求 CSR）
- `ca/intermediate/certs/intermediate.cert.pem`（由 Root 签发的 Intermediate CA 证书）
- `ca/intermediate/certs/ca-chain.cert.pem`（CA 证书链，通常为 Intermediate + Root 拼接）
- `ca/root/newcerts/*`（Root CA 已签发证书归档副本）
- `artifacts/logs/audit.jsonl`（审计日志追加）

### 删除

- 无。

### 修改

- `ca/root/index.txt`（Root 台账新增一条“签发 Intermediate”记录）
- `ca/root/serial`（Root 序列号计数器递增）

## 3. 签发服务端 + 客户端证书（`issue`）

### 生成

首次或 `server/server.cert.pem` 不存在时还会生成服务端文件：

- `server/server.key.pem`（服务端私钥）
- `ca/intermediate/csr/server.csr.pem`（服务端 CSR）
- `server/server.cert.pem`（服务端证书，由 Intermediate 签发）

每次都会生成客户端文件（以 `trainee` 为例）：

- `client/client-trainee.key.pem`（客户端私钥）
- `ca/intermediate/csr/client-trainee.csr.pem`（客户端 CSR）
- `client/client-trainee.cert.pem`（客户端证书，由 Intermediate 签发）
- `client/client-trainee.p12`（客户端导入包，含私钥/证书/链）
- `ca/intermediate/newcerts/*`（Intermediate 已签发证书归档副本）
- `artifacts/logs/audit.jsonl`（审计日志追加）

### 删除

- 无。

### 修改

- `ca/intermediate/index.txt`（Intermediate 台账新增服务端/客户端签发记录）
- `ca/intermediate/serial`（Intermediate 序列号计数器递增）

## 4. 校验证书链（`verify`）

### 生成

- `artifacts/logs/verify-chain.log`（Root→Intermediate 链验证日志）
- `artifacts/logs/verify-server.log`（服务端证书链验证日志）
- `artifacts/logs/verify-client.log`（客户端证书链验证日志）
- `artifacts/logs/audit.jsonl`（审计日志追加）
校验 1：Intermediate 是否能被 Root 信任（根到中级链）。
校验 2：server 证书是否能被 ca-chain 验证通过。
校验 3：client 证书是否能被 ca-chain 验证通过
ca-chain 主要验证“被 Intermediate 签发的证书链路”
### 删除

- 无。

### 修改

- `artifacts/logs/verify-chain.log`（重复执行会覆盖）
- `artifacts/logs/verify-server.log`（重复执行会覆盖）
- `artifacts/logs/verify-client.log`（重复执行会覆盖）
- `artifacts/logs/audit.jsonl`（追加）

## 6. 一键 mTLS 验证（TLS1.2，`mtls-validate`）

> 该按钮内部包含吊销流程，会改变证书状态。

### 生成

- `artifacts/logs/mtls-server.log`（本次 mTLS 服务端标准输出）
- `artifacts/logs/mtls-server.err.log`（本次 mTLS 服务端错误输出）
- `artifacts/logs/mtls-success.log`（携带有效客户端证书的握手日志）
- `artifacts/logs/mtls-no-cert.log`（不带客户端证书的握手日志）
- `artifacts/logs/mtls-revoked.log`（吊销后再次握手日志）
- `ca/intermediate/crl/intermediate.crl.pem`（若此前不存在则先生成）
- `artifacts/logs/audit.jsonl`（审计日志追加）
1、启动临时 openssl s_server（要求客户端证书，且启用 CRL 检查）。+
2、场景 A：用有效客户端证书连接一次（应成功），写 mtls-success*.log。
3、场景 B：不带客户端证书连接一次（应失败），写 mtls-no-cert*.log。
4、调用 revoke() 吊销当前客户端证书并更新 CRL。
5、场景 C：再用“刚被吊销”的证书连接一次（应失败），写 mtls-revoked*.log。
6、输出服务端日志与审计记录。
### 删除

- 无。

### 修改

- `ca/intermediate/index.txt`（内部调用吊销后更新状态）
- `ca/intermediate/crlnumber`（可能递增）
- `ca/intermediate/crl/intermediate.crl.pem`（可能更新）
- 上述 `mtls-*.log`（重复执行会覆盖）

## 补充：Intl 只读按钮（不改文件）

- `刷新 Intl 状态`：只读取状态，不写入文件。
- `查看 Intl 审计`：只读取 `artifacts/logs/audit.jsonl`，不写入。
- `下载 Intl P12`：只下载，不写入仓库文件。
- `下载 Intl CA 链`：只下载，不写入仓库文件。
- `打开 Intl 仓库页`：只读访问，不写入仓库文件。
