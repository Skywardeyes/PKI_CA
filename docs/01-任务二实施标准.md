# 实施标准

## 1. 目标范围
- 范围限定在本地测试环境；**单人单机**、**无 Docker**。
- 完成两级 CA（Root + Intermediate）、服务端证书、客户端证书、mTLS 联调与验证；可选 Web 控制台、HTTP 证书仓库（CDP/AIA）、审计日志。
- 形成可复现文档、验证证据和汇报材料；**操作步骤、验证矩阵、现场演示时间轴**均见 [02-执行手册.md](02-执行手册.md)。
- 讲述口径按 [02-执行手册.md](02-执行手册.md) **§2.1** 统一（管理员/终端用户/Web 控制台/HTTP 证书仓库/业务 mTLS 服务）。

## 2. 命名与目录规范
- 根目录约定：
  - `ca/`：CA 配置与数据库目录
  - `configs/nginx/`：Nginx mTLS 配置
  - `scripts/`：可选历史脚本（**不作为 Web 运行时依赖**；控制台编排见 `web/app/services/pki_workflow.py`）
  - `client/`：客户端证书导出
  - `artifacts/`：验证日志、截图占位、命令输出
  - `docs/`：实施标准、执行手册（含验证矩阵与演示时间轴）、汇报提纲等
  - `web/`：Web PKI 控制台（主操作入口，见 `web/README.md`）
- 证书文件命名：
  - RootCA：`ca.cert.pem`、`ca.key.pem`
  - IntermediateCA：`intermediate.cert.pem`、`intermediate.key.pem`
  - 服务端：`server.cert.pem`、`server.key.pem`
  - 客户端：`client-<username>.cert.pem`、`client-<username>.key.pem`
  - 证书链：`ca-chain.cert.pem`
- 示例 DN：
  - Root：`C=CN, ST=Shanghai, O=Koal, OU=Training, CN=Koal Root CA`
  - Intermediate：`C=CN, ST=Shanghai, O=Koal, OU=Training, CN=Koal Intermediate CA`
  - Server：`C=CN, ST=Shanghai, O=Koal, OU=PKI, CN=localhost`
  - Client：`C=CN, ST=Shanghai, O=Koal, OU=PKI, CN=<username>`

## 3. 证书策略
- RootCA：有效期 3650 天，仅签发 IntermediateCA。
- IntermediateCA：有效期 1825 天，签发服务端/客户端证书。
- Server 证书：有效期 825 天，必须包含 SAN（`DNS:localhost, IP:127.0.0.1`），EKU = `serverAuth`。
- Client 证书：有效期 825 天，EKU = `clientAuth`。
- 必须启用 CRL，并验证吊销证书后拒绝访问。
- 演示环境：由 Intermediate 签发的叶子证书应包含 **CDP**（CRL 分发点）与 **AIA（caIssuers）**，URI 固定为 `http://127.0.0.1:8765/repo/...`，与 Web 控制台同机发布；若更改端口或主机名，须同步修改 [ca/openssl-intermediate.cnf](../ca/openssl-intermediate.cnf) 并重新签发叶子证书。

## 4. 安全基线
- 私钥权限最小化（仅所有者可读写）。
- RootCA 私钥不参与在线服务。
- 客户端 P12 文件设置导出密码，禁止明文共享。
- 日志中避免打印私钥内容。

## 5. 验证基线
- 成功场景：
  - 证书链校验通过。
  - 携带合法客户端证书访问成功。
- 失败场景：
  - 无客户端证书访问失败。
  - 非信任链客户端证书访问失败。
  - 已吊销客户端证书访问失败。

## 6. 交付基线
- 必须交付：
  - CA 配置模板（`ca/openssl-*.cnf`）与可复现的 **Python 工作流**（`web/app/services/pki_workflow.py` 等）；`scripts/` 为可选对照物。
  - 验证日志与证据记录（`artifacts/logs/`，含 `verify-*.log`；使用 Web 时含 `audit.jsonl`）。
  - PKI 基础知识汇报材料（见 [04-PKI基础知识汇报提纲.md](04-PKI基础知识汇报提纲.md)）。
- Web 控制台应作为主操作入口，并能通过按钮触发全部核心工作流（演示重置 / 初始化 / 建 CA / 签发 / 吊销 / 校验 / 一键 mTLS 验证）。
- 执行与验收以 [02-执行手册.md](02-执行手册.md) 为准（含第 14 节验收清单、第 16 节验证矩阵、第 17 节演示时间轴）。
